Random HTTP Status Code API

A service that can returns a random, a given or a list of HTTP Status Codes.

Fabio Marciano

What is it?

The Random HTTP Status Code API is an apllication that can return a random (or a given) status response or a list of HTTP Status Codes.

How it works?

Simple make a request using any of one http methods (GET, POST, PUT, DELETE and etc.) to the proper endpoint to get a random response.

Alternativelly, you can specify the code you wish on your request, you will get the same type of response (if applicable).

The application can also provide a list of all HTTP Status Codes (json format).

Endpoints

Currently the API provide three endpoints: random response, fixed response and list of all codes.

Random response

In order to obtain a random response you can use the following request (using GET method, for example):

Request:
GET HTTP/1.1 /api
Host:
https://random-http-status-code-api.onrender.com/

The code above can return, randomically, any one of the enabled HTTP Status Codes. Example:

Response:
HTTP/1.1 410 Gone

Expected response

If one of the enabled status codes is given on the request, the response will be of the same category. Example:

Request:
POST HTTP/1.1 /api/500
Host:
https://random-http-status-code-api.onrender.com/
Response:
HTTP/1.1 500 Internal Server Error

List

The API can also provide a list of all enabled HTTP Status Codes using the following request:

Request:
GET HTTP/1.1 /api/list
Host:
https://random-http-status-code-api.onrender.com/
Response:
HTTP/1.1 200 Ok
Content-Type:
application/json
[
  {
    "code": "100",
    "message": "Continue"
  },
  {
    "code": "101",
    "message": "Switching Protocols"
  },
  ...
]
